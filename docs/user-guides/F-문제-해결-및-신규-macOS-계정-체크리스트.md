# F. 문제 해결 및 신규 macOS 계정 체크리스트

이 문서는 `깨끗한 신규 macOS 사용자 계정`에서 `homepage`를 처음 설치한 사람 기준으로,
어디에서 막힐 가능성이 큰지 먼저 정리한 점검 문서입니다.

기준 사용자:

- 새 macOS 사용자 계정
- Obsidian도 막 설치한 상태
- 빈 vault 또는 새로 만든 수업용 vault
- 기존 `homepage` 설정/캐시/샘플 노트가 없음
- BRAT 설치 경험이 없을 수 있음
- Google Form / Dataview / Templater / QuickAdd도 아직 안 써봤을 수 있음

## 1. 가장 위험한 숨은 전제 우선순위

### P0. 설치는 됐지만 “이제 뭘 눌러야 하는지”가 보이지 않음

증상:

- 플러그인을 켰는데 바로 화면이 변하지 않음
- 커맨드 팔레트를 모르면 시작 지점을 못 찾음
- 리본 아이콘 의미를 모르면 첫 실행 자체를 놓침

왜 위험한가:

- 개발자/기존 사용자에게는 익숙하지만, 신규 사용자는 “설치 완료 = 바로 화면이 나와야 한다”고 기대하기 쉽습니다.
- 현재 구조는 기능은 있으나, 첫 실행 안내 흐름이 약합니다.

우선 확인:

1. 플러그인 활성화 직후, 초보자가 따라갈 첫 행동이 눈에 띄는가
2. 설정 화면에 `1단계, 2단계, 3단계` 식 버튼형 흐름이 있는가
3. 커맨드 팔레트를 몰라도 시작 가능한가

### P0. starter pack이 사실상 숨은 의존성인데 릴리즈 사용자에게 같이 전달되지 않음

증상:

- README/설치 안내에는 플러그인 설치만 적혀 있음
- 실제 운영에 필요한 템플릿/예시/샘플 워크플로우는 사용자가 따로 만들어야 함
- 작성자 계정에서는 이미 있던 노트/폴더/운영 습관 덕분에 문제가 가려짐

왜 위험한가:

- 신규 사용자는 “기능은 있는데 실제로 바로 쓸 수는 없는 상태”를 가장 답답하게 느낍니다.
- 학급관리/수업용 도구는 첫날 바로 쓸 예시가 핵심입니다.

우선 확인:

1. 플러그인만 설치해도 최소 템플릿/예시/시작 문서가 vault 안에 들어오는가
2. 교사가 첫날 바로 복붙해 쓸 수 있는 샘플 노트가 있는가
3. Google Apps Script 예시까지 포함돼 있는가

### P0. vault 상대경로가 아니라 Finder 절대경로를 붙여 넣는 경우

증상:

- 사용자가 Finder에서 복사한 `/Users/계정명/.../Vault/...` 경로를 설정에 넣음
- 다른 계정으로 옮기면 경로가 달라짐
- vault 안에 `Users/...` 같은 잘못된 폴더가 생길 수 있음

왜 위험한가:

- 초보자는 “경로”라고 하면 Finder 전체 경로를 붙여 넣는 경우가 많습니다.
- 기존 작성자 계정에서는 경로가 우연히 맞아 보일 수 있습니다.

우선 확인:

1. `홈페이지 노트 경로`, `뉴스읽기 폴더`가 절대경로 입력을 자동 보정하는가
2. 다른 계정으로 vault를 옮겼을 때 저장된 경로가 현재 vault 기준으로 다시 해석되는가
3. 경로가 잘못되면 조용히 실패하지 않고 안전한 기본값으로 돌아가는가

### P1. 외부 파일 절대경로가 사용자 홈 디렉토리에 묶임

증상:

- 예전 계정에서 설정한 배경 이미지가 새 계정에서 안 보임
- `/Users/old-user/...` 형태의 경로가 남아 있음

왜 위험한가:

- 배경 이미지는 필수 기능은 아니지만, “설정은 남아 있는데 결과가 안 보이는” 상황을 만듭니다.

우선 확인:

1. 외부 이미지 경로가 없거나 깨졌을 때 앱이 정상 동작하는가
2. 현재 vault 내부 이미지를 쓰는 흐름이 더 쉬운가
3. 잘못된 외부 경로를 초보자에게 이해 가능한 문장으로 안내하는가

### P1. 첫 실행 시 링크는 보이는데 실제 노트는 아직 없는 상태

증상:

- 홈페이지에는 공지/과제/리포트 링크가 보이는데 실제 노트가 아직 없음
- 위키링크를 눌렀을 때 Obsidian 동작을 모르면 “깨진 링크”로 느껴질 수 있음

왜 위험한가:

- 기존 Obsidian 사용자는 자연스럽게 새 노트를 만들지만, 초보자는 오류처럼 느낍니다.

우선 확인:

1. 첫 실행 후 최소한 오늘 공지/기본 안내 노트는 자동으로 준비되는가
2. 링크 대신 버튼/명령 중심 흐름이 충분히 제공되는가

### P1. Google Form 연동은 선택 기능인데 처음부터 준비돼 있어야 하는 것처럼 느껴짐

증상:

- URL 입력칸이 여러 개라서 초보자가 부담을 느낌
- 당장 폼이 없어도 되는지 판단하기 어려움

왜 위험한가:

- 수업/학급관리의 핵심은 먼저 노트 운영이고, 폼 연동은 2단계 기능입니다.

우선 확인:

1. “플러그인 없이도 되는 기본 흐름”과 “폼을 쓰면 더 좋아지는 흐름”이 분리 설명돼 있는가
2. URL이 비어 있어도 첫 사용이 막히지 않는가

### P2. README가 개발자 정보와 실제 사용자 설치 정보를 섞어 보여 줌

증상:

- Node.js, npm, 테스트 명령이 먼저 보임
- 일반 교사는 개발 환경이 필요한 것으로 오해할 수 있음

왜 위험한가:

- 새 계정 사용자에게는 `설치`, `첫 실행`, `Obsidian에서 뭘 눌러야 하는지`가 먼저 보여야 합니다.

## 2. 신규 macOS 사용자 계정 기준 재현 시나리오

아래 시나리오는 “내 계정에 이미 있는 설정/캐시 덕분에 가려진 문제”를 찾기 위한 최소 재현 절차입니다.

### 시나리오 A. 완전 신규 계정 + 완전 신규 vault

1. 새 macOS 사용자 계정을 만듭니다.
2. Obsidian을 새로 설치합니다.
3. iCloud/Google Drive가 아닌 로컬 경로에 새 vault를 만듭니다.
4. Community plugins를 켭니다.
5. BRAT를 설치합니다.
6. `homepage`를 BRAT로 설치합니다.
7. 플러그인을 활성화한 직후, 사용자가 “다음 행동”을 이해하는지 확인합니다.

관찰 포인트:

- 시작 버튼/가이드가 눈에 띄는가
- 아무 설정 없이도 첫 성공 경험을 만들 수 있는가
- 초보자가 커맨드 팔레트 없이도 시작 가능한가

### 시나리오 B. 신규 계정 + 기존 vault 복사본

1. 원래 계정에서 쓰던 수업용 vault를 다른 사용자 계정으로 복사합니다.
2. `.obsidian` 폴더까지 포함해 복사합니다.
3. 새 계정에서 Obsidian으로 그 vault를 엽니다.
4. `homepage` 설정이 남아 있다면 그대로 불러옵니다.

관찰 포인트:

- `홈페이지 노트 경로`, `뉴스읽기 폴더`가 절대경로로 저장돼 있지 않은가
- 배경 이미지 같은 외부 경로가 이전 사용자 홈(`/Users/old-user`)를 가리키지 않는가
- 잘못된 경로 때문에 새 계정에서 생성 위치가 꼬이지 않는가

### 시나리오 C. 신규 계정 + Finder 절대경로 붙여 넣기

1. 설정에서 `홈페이지 노트 경로`에 Finder 전체 경로를 붙여 넣습니다.
2. 설정에서 `뉴스읽기 폴더`에도 전체 경로를 붙여 넣습니다.
3. 홈페이지 생성/공지 생성 명령을 실행합니다.

관찰 포인트:

- 현재 vault 기준 상대경로로 자동 보정되는가
- `Users/...` 폴더가 vault 내부에 잘못 생기지 않는가

### 시나리오 D. 신규 계정 + Google Form 미설정 상태

1. Google Form URL을 하나도 넣지 않습니다.
2. `학급 홈페이지 열기`, `오늘자 공지 노트 생성`, `오늘자 뉴스읽기 과제 생성`을 실행합니다.

관찰 포인트:

- 비어 있는 URL 때문에 생성이 막히지 않는가
- 안내 문구가 “나중에 넣어도 됨”이라는 메시지를 주는가

### 시나리오 E. 신규 계정 + Obsidian 플러그인 추가 활용

1. 플러그인을 하나도 설치하지 않은 상태로 기본 흐름을 실행합니다.
2. 이후 Dataview / Templater / QuickAdd를 각각 추가했을 때 더 좋아지는 지점을 확인합니다.

관찰 포인트:

- 추가 플러그인 없이도 핵심 흐름이 완성되는가
- 추가 플러그인이 있으면 어디가 더 편해지는지 문서가 분리돼 있는가

## 3. 깨끗한 신규 사용자 계정 체크리스트

아래 항목은 실제 검증 체크리스트입니다.

### 설치 전

- [ ] 새 macOS 사용자 계정이다.
- [ ] Obsidian 설정/플러그인/캐시가 없다.
- [ ] 새 vault를 로컬 경로에 만들었다.
- [ ] vault 경로에 한글/공백이 있어도 테스트한다.

### 설치

- [ ] Community plugins 활성화 경로를 초보자가 이해할 수 있다.
- [ ] BRAT 설치 방법이 그림 없이도 충분히 따라가기 쉽다.
- [ ] `homepage` 저장소 주소를 그대로 복붙해 설치할 수 있다.
- [ ] 설치 직후 플러그인 목록에 `homepage`가 보인다.

### 첫 실행

- [ ] 첫 실행 후 무엇을 눌러야 하는지 바로 보인다.
- [ ] `1단계, 2단계, 3단계` 식의 빠른 시작 흐름이 있다.
- [ ] 기본 구조 생성이 성공한다.
- [ ] 홈페이지 노트가 열린다.
- [ ] 공지/과제 생성 흐름이 바로 이어진다.

### 경로/계정 이동

- [ ] 절대경로를 붙여 넣어도 현재 vault 기준 상대경로로 바뀐다.
- [ ] 다른 계정으로 옮긴 설정이 있어도 현재 계정에서 안전하게 열린다.
- [ ] 이전 사용자 홈 디렉토리 경로 때문에 생성 위치가 꼬이지 않는다.

### Obsidian 중심 사용성

- [ ] 플러그인 없이도 쓸 수 있는 기본 폴더 구조 예시가 있다.
- [ ] 복붙 가능한 템플릿이 vault 안에 있다.
- [ ] 샘플 노트가 있어서 초보자가 “완성본 모양”을 먼저 볼 수 있다.
- [ ] Dataview / Templater / QuickAdd는 선택 사항으로 분리 설명된다.

### Google Apps Script / Google Form

- [ ] URL이 없어도 기본 사용이 막히지 않는다.
- [ ] URL을 어디에 넣는지 설정 화면에서 바로 이해된다.
- [ ] 초보자용 Google Apps Script 복붙 예시가 있다.
- [ ] 권한 허용 방법과 흔한 오류가 문서에 정리돼 있다.

### 문서

- [ ] 설치 문서와 개발 문서가 분리돼 있다.
- [ ] “교사용 / 초보자용” 문체로 작성돼 있다.
- [ ] 클릭 위치, 파일 위치, 복붙 위치가 구체적으로 적혀 있다.

## 4. 현재 기준 우선 수정 권장 순서

1. 첫 실행 빠른 시작 흐름을 설정 화면과 명령에서 바로 보이게 만들기
2. starter pack 성격의 샘플/템플릿/안내 노트를 vault 안에 넣는 기능 제공
3. 절대경로 입력 및 다른 계정에서 가져온 설정을 현재 vault 기준으로 자동 보정
4. README를 개발자용과 실사용자용으로 분리 정리
5. Obsidian 중심 문서, Google Apps Script 복붙 문서, 문제 해결 문서를 별도 정리
